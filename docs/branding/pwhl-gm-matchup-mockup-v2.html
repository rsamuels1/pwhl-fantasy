Design Notes — My Matchup v2

Breakpoints

  • >900px (desktop): Two-column grids for top performers, roster breakdown, league leaders. Hero scores in three-column layout. Full nav visible.
  • 600–900px (tablet): Two-column grids collapse to single column. Hero scores stack vertically. Nav compresses to show only first 4 tabs.
  • <600px (mobile): Full single column. Hero font size clamps down. Nav is hidden (relies on mobile bottom nav from league layout). Cards have reduced padding (16px). Roster tables scroll horizontally.

State variants shown in this mockup

  • Active week / DuelHero: The primary state — playoff 1v1, currently winning. Shows: recap, alert, hero with live pulse, storyline, playing tonight, swing players, performers, league leaders, roster tables, rival H2H, activity.
  • VTF FieldHero (Variant A): Regular season mode. Score on the left; W-L record vs field on the right; full 8-team standings table below scores. No opponent roster card shown (VTF has no single opponent).
  • Upcoming week (Variant B): Between scoring periods. Amber nudge strip shown. Hero uses em-dashes for scores. Win probability bar hidden. CTA button replaces live score context.

Visual decisions

  • DM Mono for scores: Monospaced numeric font prevents score width jumping as digits change during live updates. Tabular-nums fallback for body text.
  • Syne for wordmark only: High-weight geometric sans creates authority without competing with data-dense content.
  • No emoji in section headers: Per brand brief — professional sports management aesthetic, not arcade fantasy. Emoji retained only in the storyline chip and recap strip where personality is intentional.
  • Slot badge color system: F=blue / D=green / G=amber / UTIL=purple / BN=muted. Consistent with lineup page and draft room so managers develop muscle memory across the app.
  • Games-remaining badge: Red "0" is urgent; indigo "NG" is informational. Only shown on MY team's roster (opponent games badge is irrelevant to the manager's decision-making).
  • Stat breakdown chips: Only shown on my team's active players in the roster table. Compact pills communicate what happened without a full stat table. Opponent chips omitted — focus stays on my team's performance.
  • Subtle diagonal grid texture in hero: Evokes scoreboards and arena jumbotrons without being literal. Very low opacity so it doesn't compete with numeric content.
  • 44px minimum touch targets: All pill-links, tab buttons, and nav links meet this threshold for mobile usability.

Missing from this mockup (in-code features not shown)

  • Champion card (gold-tinted, shown only when playoffStatus=COMPLETE and I won)
  • Eliminated card (shown when I lost in playoffs and no more matchups)
  • Playoff-pending card (between playoff rounds)
  • Replay simulator controls (only visible to commissioners/founders on replay leagues)
  • InlineLineupEditor (replaces read-only RosterTable in the upcoming state — click-to-swap directly on this page)
  • LiveScoreRefresh client component (triggers router.refresh() every 60s with "Live · updated Ns ago" pulse)
