PWHL Fantasy  ·  Product Roadmap  ·  Updated June 12, 2026

The road to a public beta. Alignment, then features — eight phases, thirty-three features, MVP by late October 2026.

The flagship experience is the live fantasy season — drafting real players, setting weekly lineups, and competing in matchups scored from real PWHL games. This roadmap merges two tracks: aligning the code to the approved v1 rules (Phase 0, launch blockers) and the feature builds (Phases 1–7). Below: the readiness scorecard, every feature, the sprint plan, and a projected launch date.

    Rename "Projected" → "Matchup Proj", hide "This week" between weeks. Single-component edit.

    ~40K
  4. Standard — one feature per session (60–90K tokens)
  5. Error Handling · #4
    Empty + loading states across all core pages. Many files, but each change is small and localized. Beta prerequisite.
    ~65K
  6. Weekly Performance Dashboard · #29
    New page replacing the Schedule tab — week-by-week rankings + position-group breakdown. No schema changes.
    ~65K
  7. Mobile Optimization · #3
    Responsive CSS across draft room, matchup, and roster screens. Beta prerequisite.
    ~75K
  8. Team Analysis & Insights · #25
    New Analysis tab — position-group trends, FA recommendations, trade suggestions once #7 lands.
    ~85K
  9. Heavy lifts — plan a fresh session (100K+ tokens)
  10. League Onboarding · #2
    Entirely unbuilt — welcome flow, setup wizard, draft guide. Beta prerequisite despite token cost.
    ~100K

    Infrastructure first — records every add, drop, waiver, and trade. Built on CT-002 audit log from Sprint 2.
    ~55K
  12. Trade System · #7
    New domain: schema tables, API routes, proposal / review / approval UI. Built on top of Transaction History. Plan a dedicated session.

    ~110K
