# League Homepage v1 Specification

Version: 1.0

Priority: P2

Purpose: Define the primary landing experience when a manager enters a league.

---

# Design Goal

A manager should understand the current state of their league within 10 seconds.

The homepage should answer:

1. How is my team doing?
2. Who am I playing?
3. Where do I rank?
4. What action do I need to take?
5. What happened recently?

---

# Information Priority

## Tier 1

Must appear above the fold.

### Current Matchup Card

Display:

- Opponent
- Current score
- Matchup status
- Win probability (future)

Purpose:

Most important league information.

---

### Lineup Status

Display:

- Lineup complete
- Missing players
- Locked status

Primary CTA:

"Set Lineup"

---

### Standings Snapshot

Display:

- Top 5 teams
- User rank
- VP totals

---

# Tier 2

### League Activity Feed

Examples:

- Draft completed
- Manager joined
- Trade completed
- Commissioner announcement

---

### Upcoming Deadlines

Examples:

- Lineup lock
- Draft start
- Playoff start

---

# Tier 3

### League News

Future enhancement.

### Featured Players

Future enhancement.

---

# Mobile Layout

Order:

1. Matchup
2. Lineup
3. Standings
4. Activity
5. Deadlines

---

# Desktop Layout

Left:

Matchup

Center:

Standings

Right:

Lineup + Deadlines

Bottom:

Activity Feed

---

# MVP Success Criteria

User can answer:

- Who am I playing?
- What is my score?
- What is my rank?
- Do I need to set my lineup?

within 10 seconds of loading the page.
