# PWHL Fantasy Product Roadmap

This document has been split into three focused files to reduce per-session token cost.

| File | Load when... |
|---|---|
| [roadmap-index.md](roadmap-index.md) | You need a current-state briefing, the MVP scorecard, or the "what to build next" queue |
| [roadmap-features.md](roadmap-features.md) | You're updating or reviewing a specific feature card (Phases 0–7) |
| [roadmap-sprints.md](roadmap-sprints.md) | You're planning or reviewing a sprint, or need the launch timeline |

---

## Why the split?

The original `roadmap.md` was 1,320 lines. Most work only needs one section:
- **Sprint reviews** → load `roadmap-index.md` + `roadmap-sprints.md` (~380 lines, ~50% reduction)
- **Feature status updates** → load `roadmap-features.md` only (~900 lines)
- **Orientation briefings** → load `roadmap-index.md` only (~100 lines)

The split makes each file purpose-built, reducing context cost per session while keeping all information accessible.
