# Support Playbook

Version: 1.0

Purpose: Establish operational procedures for handling common support requests.

---

# Severity Levels

## P0

League cannot continue.

Examples:

- Draft cannot start
- Standings incorrect
- Champion incorrect

Response:

Immediate

---

## P1

Major functionality broken.

Examples:

- Lineups not saving
- Scoring delays
- Commissioner tools unavailable

Response:

Same day

---

## P2

Usability issue.

Examples:

- Confusing UI
- Missing help text

Response:

Next sprint

---

# Draft Issues

## User Missed Pick

Expected behavior:

Auto-pick.

No support action required.

---

## User Disconnected

Expected behavior:

Reconnect.

Auto-pick protects draft progress.

---

## Draft Frozen

Escalate immediately.

Potential P0.

---

# Lineup Issues

## Player Locked

Expected behavior.

Game already started.

No support action required.

---

## Lineup Not Saving

Verify:

- User logged in
- Roster valid

Escalate if reproducible.

---

# Scoring Issues

Verify:

1. Stat source
2. League scoring rules
3. Calculation engine

Escalate confirmed discrepancies.

---

# Commissioner Requests

## Replace Manager

Allowed.

Verify commissioner identity.

---

## Undo Transaction

Document:

- Requestor
- Reason
- Timestamp

---

## Force Roster Move

Requires audit log entry.

---

# Playoff Issues

Verify:

- Standings
- VP totals
- Tiebreakers

Escalate if qualification appears incorrect.

---

# Escalation Matrix

P0 → Engineering immediately

P1 → Same-day review

P2 → Product backlog

P3 → Documentation improvement

---

# Support Metrics

Track:

- Ticket volume
- Resolution time
- Repeat issues
- Root causes

Goal:

Reduce recurring tickets through better product design.
