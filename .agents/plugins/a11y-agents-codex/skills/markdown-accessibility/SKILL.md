# Markdown Accessibility Router

Use this skill for markdown documentation and publication workflows.

## Workflow

1. Check heading hierarchy, descriptive links, image alternatives, table structure, code block labeling, diagrams, and generated anchor links.
2. Check installed Accessibility Agents extensions for editorial standards, company vocabulary, legal disclaimers, or publishing rules.
3. Load specialist references only when needed.
4. Return concrete edits or review findings with line references when possible.

For repository publication, follow this repository's no-emoji policy.
