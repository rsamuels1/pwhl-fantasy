# Developer Tools Router

Use this skill for desktop, Python, scanner, and developer-tool accessibility work.

## Workflow

1. Identify the platform: Python CLI, wxPython, desktop GUI, NVDA add-on, scanner tooling, CI integration, or MCP/server tooling.
2. Check installed Accessibility Agents extensions for company tooling standards, internal scanner rules, or platform-specific requirements.
3. Use the matching Codex subagent for focused review when subagents are requested.
4. Keep recommendations practical: APIs, keyboard behavior, screen reader naming, focus handling, test coverage, and packaging impact.

Extension rules should supplement core accessibility standards, not hide them.
