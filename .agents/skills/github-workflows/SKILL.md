# GitHub Workflows Router

Use this skill for GitHub workflow tasks that should preserve accessible repository operations.

## Workflow

1. Identify the GitHub workflow: PR review, issue triage, Actions, releases, projects, wiki, security alerts, notifications, or repository setup.
2. Check installed Accessibility Agents extensions for organization-specific labels, required checks, release standards, or compliance reporting.
3. Use `pr-review` for code review and add domain specialists when the diff touches accessibility-sensitive surfaces.
4. Keep findings concrete, ordered by risk, and tied to files, checks, or GitHub objects.

Do not replace GitHub product policy or repository ownership decisions. Surface risks and suggested actions.
